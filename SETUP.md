# Kubernetes The Hard Way — Local Setup

Automates [kubernetes-the-hard-way](https://github.com/kelseyhightower/kubernetes-the-hard-way) using **Terraform** (VMs) and **Ansible** (cluster config) on your local machine via LXD.

## Requirements

- Ubuntu / Debian host (or any Linux with snapd)
- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.0
- Your SSH public key at `~/.ssh/id_rsa.pub` (or set `ssh_public_key_path` in variables)

## Steps

### 1. Clone the repo

```bash
git clone https://github.com/AyoubJedidi/k8s-automation.git
cd k8s-automation
```

### 2. Install and initialize LXD

```bash
chmod +x setup.sh
./setup.sh
```

This installs LXD via snap and runs `lxd init --auto` if not already done.

### 3. Provision the VMs

```bash
cd terraform
terraform init
terraform apply
```

This creates 4 local VMs:

| VM       | IP             | Role            |
|----------|----------------|-----------------|
| jumpbox  | 10.142.131.10  | Admin / bastion |
| server   | 10.142.131.11  | Control plane   |
| node-0   | 10.142.131.12  | Worker node     |
| node-1   | 10.142.131.13  | Worker node     |

### 4. Verify SSH access

```bash
ssh root@10.142.131.10   # jumpbox
ssh root@10.142.131.11   # server
ssh root@10.142.131.12   # node-0
ssh root@10.142.131.13   # node-1
```

### 5. Tear down

```bash
terraform destroy
```

## Notes

- All VMs use Ubuntu 22.04 and get static IPs configured at boot via cloud-init.
- Your SSH public key (`~/.ssh/id_rsa.pub`) is added to `root` on every VM.
- A dedicated jumpbox SSH keypair is generated by Terraform and saved to `terraform/jumpbox_id_ed25519`.
- The Ansible inventory is auto-generated at `ansible/inventory/hosts.ini` after apply.
