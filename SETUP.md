# Kubernetes The Hard Way — Local Setup

Automates [kubernetes-the-hard-way](https://github.com/kelseyhightower/kubernetes-the-hard-way) using **Terraform** (VMs) and **Ansible** (cluster config) on your local machine via LXD.

## Requirements

- Ubuntu / Debian host (or any Linux with snapd)
- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.0
- Your SSH public key at `~/.ssh/id_rsa.pub` (or set `ssh_public_key_path` in variables)
- Your user must be in the `lxd` group (the setup script handles this)
- `/snap/bin` must be in your `PATH` (the setup script handles this)

## Steps

### 1. Clone the repo

```bash
git clone https://github.com/AyoubJedidi/k8s-automation.git
cd k8s-automation
```

### 2. Install and initialize LXD

```bash
chmod +x setup.sh
./setup.sh
```

This script:
- Installs LXD via snap if not already installed
- Adds your user to the `lxd` group (required for socket access)
- Runs `sudo lxd init --auto` to create the default storage pool if none exists
- Creates the `lxdk8s` bridge network (`10.142.131.1/24`) used by all VMs
- Adds `/snap/bin` to your `~/.bashrc` so `lxc` is available in PATH

> **Note:** If the script prints a warning about the `lxd` group, you must log out and log back in (or run `newgrp lxd` in your current shell), then **re-run `./setup.sh`** before proceeding. The bridge network is created on the second run.

### 3. Provision the VMs

```bash
cd terraform
terraform init
terraform apply
```

This creates 4 local VMs:

| VM       | IP             | Role            |
|----------|----------------|-----------------|
| jumpbox  | 10.142.131.10  | Admin / bastion |
| server   | 10.142.131.11  | Control plane   |
| node-0   | 10.142.131.12  | Worker node     |
| node-1   | 10.142.131.13  | Worker node     |

### 4. Verify SSH access

```bash
ssh root@10.142.131.10   # jumpbox
ssh root@10.142.131.11   # server
ssh root@10.142.131.12   # node-0
ssh root@10.142.131.13   # node-1
```

> **Note:** If you get a `WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED` error after re-provisioning, clear the old keys:
> ```bash
> ssh-keygen -R 10.142.131.10 -R 10.142.131.11 -R 10.142.131.12 -R 10.142.131.13
> ```

From the jumpbox, SSH to other VMs using the auto-provisioned key:
```bash
ssh root@10.142.131.11   # no password needed — key is pre-installed
```

### 5. Tear down

```bash
terraform destroy
```

## Notes

- All VMs use Ubuntu 22.04 and get static IPs configured at boot via cloud-init.
- Your SSH public key (`~/.ssh/id_rsa.pub`) is added to `root` on every VM.
- A dedicated jumpbox SSH keypair is generated by Terraform and saved to `terraform/jumpbox_id_ed25519`.
- The Ansible inventory is auto-generated at `ansible/inventory/hosts.ini` after apply.

## Troubleshooting

### `Error: LXD socket with write permissions not found`

```
│ Error: LXD socket with write permissions not found.
│ Searched LXD directories: [/var/lib/lxd /var/snap/lxd/common/lxd]
```

This means the Terraform LXD provider cannot access the LXD Unix socket. Common causes:

**1. Your user is not in the `lxd` group**

```bash
sudo usermod -aG lxd $USER
newgrp lxd   # apply in current shell without logout
```

Or log out and log back in, then retry `terraform apply`.

**2. `/snap/bin` is not in your PATH**

LXD is installed via snap, so its binaries live in `/snap/bin`. Verify:

```bash
/snap/bin/lxc list   # should work
lxc list             # fails if /snap/bin is not in PATH
```

Fix permanently:

```bash
echo 'export PATH=$PATH:/snap/bin' >> ~/.bashrc
source ~/.bashrc
```

**3. LXD is not initialized**

```bash
sudo /snap/bin/lxd init --auto
```

---

### `Error: No root device could be found`

```
Failed getting root disk: No root device could be found
```

LXD has no storage pools configured. Run `setup.sh` — it detects this and calls `sudo lxd init --auto` to create the `default` pool automatically.

To verify manually:
```bash
sudo /snap/bin/lxc storage list
```

If the list is empty, run:
```bash
sudo /snap/bin/lxd init --auto
```

---

### `Error: Network not found` (lxdk8s)

```
Failed to load network "lxdk8s" for project "default": Network not found
```

The `lxdk8s` bridge was not created. This happens if you skipped re-running `setup.sh` after the initial group warning. Run:

```bash
./setup.sh
```

Or create it manually:
```bash
lxc network create lxdk8s ipv4.address=10.142.131.1/24 ipv4.nat=true ipv6.address=none
```

---

### VMs get random DHCP IPs instead of static ones

If `lxc list` shows IPs like `10.142.131.180` instead of `10.142.131.10`, cloud-init's default netplan (`50-cloud-init.yaml`) is overriding the static config. This is handled automatically by Terraform on fresh deploys. If you hit it on existing VMs:

```bash
for vm in jumpbox server node-0 node-1; do
  lxc exec $vm -- rm -f /etc/netplan/50-cloud-init.yaml
  lxc exec $vm -- netplan apply
done
```

---

### SSH from jumpbox to other VMs fails

The jumpbox SSH keypair is auto-generated by Terraform and written to `~/.ssh/id_ed25519` on the jumpbox. If SSH fails:

- Check the key exists: `lxc exec jumpbox -- ls ~/.ssh/`
- If missing, re-run `terraform destroy && terraform apply`
- Do not use a password — use the key directly: `ssh root@10.142.131.11`
